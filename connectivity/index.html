      Glossary — strong connectivity, DFS, lowlink, condensation
      Strongly connected component (SCC)
      A maximal set of vertices in a directed graph in which every vertex can reach every other. Two vertices are in the same SCC iff there is a directed path each way. Each SCC is shaded a distinct colour.
      Condensation
      Contract every SCC to a single node; the result is always a directed acyclic graph. SCC decomposition is the first step of many directed-graph algorithms.
      DFS tree, finishing time
      A depth-first search builds a forest of tree edges; a vertex finishes when its whole subtree is explored. Kosaraju orders the second pass by decreasing finishing time.
      Back / cross edge
      A non-tree edge to an already-discovered vertex. Whether it stays "inside" the current component decides where SCC boundaries fall. Shown in gold when it updates a lowlink (Tarjan) or pops a boundary (Gabow).
      Transpose graph
      The same graph with every edge reversed. Kosaraju runs its second DFS on the transpose; the visualization flips all arrowheads during pass 2.
      Kosaraju
      KosarajuStrongConnectivityInspector: (1) DFS the graph, pushing vertices by finishing time; (2) DFS the transpose in decreasing finishing-time order — each tree is one SCC. Two passes, O(V+E). Follows CLRS §22.5.
      Lowlink (Tarjan)
      In a single DFS, each vertex gets a discovery index and a low = the smallest index reachable via its subtree plus one back edge. When a vertex's low equals its own index, it is an SCC root: pop the stack down to it. The tag shows disc/low.
      Tarjan vs JGraphT
      Classic Tarjan is shown here for teaching. JGraphT ships GabowStrongConnectivityInspector — Cheriyan-Mehlhorn/Gabow's path-based variant — rather than the lowlink form; both are single-pass and linear-time.
      Gabow (path-based)
      GabowStrongConnectivityInspector: one DFS with two stacks — S holds all not-yet-assigned vertices, B marks component boundaries. A back edge pops B down to the target's position (contracting a cycle); when the top of B is the current vertex, the top of S down to it is an SCC. No lowlink arithmetic. The tag shows the S position.
      Same answer, different bookkeeping
      All three return the identical SCC partition. They differ in passes (2 vs 1 vs 1), whether they build the transpose (Kosaraju only), and what they track (finishing order / lowlink numbers / two stacks).
      What you're watching — three SCC algorithms

      Pick any two algorithms with the A and B selectors

      Three strongly-connected-component algorithms run on the same directed graph; two are JGraphT classes from org.jgrapht.alg.connectivity:

      • Kosaraju — KosarajuStrongConnectivityInspector. Pass 1 DFS records finishing times; the graph is then transposed (arrowheads flip) and pass 2 DFS in decreasing finishing order carves out one SCC per tree.
      • Tarjan — classic lowlink algorithm (shown for teaching). One DFS; each vertex tracks disc/low; an SCC is popped off the stack when a vertex's low equals its own discovery index.
      • Gabow — GabowStrongConnectivityInspector. One DFS with two stacks; back edges pop the boundary stack to contract cycles, and an SCC is sliced off the main stack when the current vertex reaches the top of the boundary stack.

      The punchline: the colours at the end are identical — the same SCC partition — but Kosaraju needs two passes and the transpose, while Tarjan and Gabow each finish in a single pass with different stack bookkeeping. The two panes run independently; total progress is the slower pane's step count.
