    Glossary — Hamiltonian path, topological order, longest-path DP
    Hamiltonian path
    A path in a graph that visits every vertex exactly once. Unlike a Hamiltonian cycle, it does not have to return to its start vertex. Deciding existence is NP-complete in general — but on a DAG it reduces to longest path, which is polynomial.
    Directed acyclic graph (DAG)
    A directed graph with no directed cycle. Equivalently, its vertices admit a topological order: a linear arrangement v₁, v₂, …, vₙ in which every edge u → v has u appear before v.
    Topological order
    Computed in O(|V| + |E|) by Kahn's algorithm: repeatedly emit a vertex with in-degree 0 and remove its outgoing edges. If at any point no zero-in-degree vertex exists while vertices remain, the graph has a cycle and Hamiltonian-path-via-DAG is inapplicable.
    Longest-path DP on a DAG
    Process vertices in topological order. For each vertex v, set L(v) = 1 + max{L(u) : (u → v) ∈ E} (or 1 if no incoming edge). Remember the argmax as pred(v) for reconstruction. Total time O(|V| + |E|).
    Hamiltonian-path test
    Once the DP finishes, let L* = max_v L(v). The DAG has a Hamiltonian path iff L* = |V| — the longest path is forced to visit every vertex. The path itself is reconstructed by following pred backwards from the argmax vertex.
    Predecessor pointers
    During the DP, every time we update L(v) via an incoming edge (u → v), we also store pred(v) = u. After the DP, reconstruction is a simple backward walk along these pointers from the best end vertex.
    Why this only works on DAGs
    In a graph with a cycle, "longest simple path" is NP-hard. The DP above implicitly assumes that processing v after all its predecessors is safe — which is only true when no edge ever points "backwards" in the iteration order. Topological order guarantees that.
    What you're watching — DAG longest-path Hamiltonian solver

    DagHamiltonianPath

    For a directed acyclic graph the Hamiltonian-path problem collapses from NP-complete to O(|V| + |E|). The algorithm runs in three phases, animated step by step in the panel above:

    1. Cycle check. Compute a topological order with Kahn's algorithm. If a cycle is present, the algorithm rejects the input — visualised as the unprocessable subset of vertices turning red.
    2. Longest-path DP. Sweep the topological order. For each vertex v, examine every incoming edge (u → v) and update L(v) = max(L(v), L(u) + 1), recording pred(v) = u whenever the maximum improves. The "vertex being processed" highlight tracks the sweep pointer; "edge being considered" highlights the incoming edge currently being relaxed.
    3. Existence test + reconstruction. Look at L* = max_v L(v). If L* < |V|, no Hamiltonian path exists. If L* = |V|, backtrack along pred from the argmax to recover the path, which then renders in gold on the canvas and in the "Reconstructed path" list below.

    The general Hamiltonian-path problem stays NP-complete; this is one of the cleanest examples of how a structural restriction (here, acyclicity) can eliminate the exponential blowup.
