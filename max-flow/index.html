      Glossary — flow networks, residual graph, augmenting paths, level graph, min cut
      Flow network
      A directed graph with a non-negative capacity on each edge, a source s and a sink t. A flow assigns each edge a value between 0 and its capacity; flow in equals flow out at every vertex except s and t. Edge labels show flow / capacity.
      Maximum flow
      The largest total flow that can be routed from s to t. Both algorithms here compute the same value — that's the whole point of the side-by-side.
      Residual graph
      For each edge u→v with flow f and capacity c: a forward residual arc u→v with spare capacity c − f, and a backward residual arc v→u with capacity f. Pushing along a backward arc cancels existing flow — this is how the algorithm corrects earlier greedy mistakes.
      Augmenting path
      A path from s to t in the residual graph. Its bottleneck (smallest residual capacity along it) is pushed as new flow. Repeat until no augmenting path remains — then the flow is maximum.
      Reverse push
      When an augmenting path uses a backward residual arc, it reduces flow on the original edge. Shown in red on the path. (On the small examples here Edmonds-Karp's shortest-path rule usually avoids them; the mechanism is always available.)
      Edmonds-Karp
      EdmondsKarpMFImpl: Ford-Fulkerson where the augmenting path is always a shortest one, found by breadth-first search in the residual graph. Guarantees termination in O(V·E) augmentations, total O(V·E²).
      Level graph (Dinic)
      A BFS from s labels every vertex with its distance (level). The level graph keeps only residual arcs that go from level k to level k+1. Shown by node tint and a small level tag.
      Blocking flow (Dinic)
      Within one level graph, push flow along level-respecting paths until every s→t route is saturated somewhere — a blocking flow. A DFS with a current-arc pointer finds these paths without revisiting dead ends. Then rebuild the level graph and repeat.
      Dinic's complexity
      DinicMFImpl: at most V − 1 level-graph phases, each blocking flow in O(V·E), giving O(V²·E) — and far fewer phases than Edmonds-Karp augmentations on layered graphs.
      Min cut
      A partition of vertices into S (containing s) and T (containing t); its capacity is the total capacity of edges from S to T. S is exactly the set of vertices still reachable from s in the final residual graph.
      Max-flow min-cut theorem
      The maximum flow value equals the minimum cut capacity. Watch the finale: the highlighted cut edges are all saturated, and their capacities sum to the max-flow value.
      What you're watching — two max-flow algorithms

      Pick either algorithm in each pane

      Two maximum-flow algorithms from JGraphT's org.jgrapht.alg.flow package run on the same network:

      • Edmonds-Karp — EdmondsKarpMFImpl. Each round runs a breadth-first search in the residual graph for a shortest augmenting path, then pushes its bottleneck. Simple, but it can take many separate augmentations.
      • Dinic — DinicMFImpl. Each phase builds a BFS level graph (node tints), then saturates it with a blocking flow found by DFS. On layered networks it finishes in a handful of phases where Edmonds-Karp needs one augmentation per path.

      Both finish by computing the minimum cut: the source side S is shaded green, the sink side grey, and the cut edges glow orange. Their capacities add up to the maximum flow — the max-flow min-cut theorem, on screen. The two panes run independently; total progress is the step count of the slower pane.

      JGraphT's Edmonds-Karp additionally tries to discover several shortest paths per BFS; for clarity this visualization augments one shortest path per BFS round.
