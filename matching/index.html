      Glossary — matchings, augmenting paths, maximal vs maximum, Hopcroft-Karp
      Bipartite graph
      Vertices split into two sides (left / right) with edges only between sides. Drawn as two columns; left strokes are blue, right strokes purple.
      Matching
      A set of edges with no shared endpoint — each vertex is matched to at most one partner. Matched edges are gold; matched vertices fill green.
      Maximal vs maximum
      Maximal: no edge can be added without a conflict (a local dead end). Maximum: the largest possible matching (global optimum). Greedy gives a maximal matching that can be as small as half the maximum — watch the greedy-trap example.
      Free (exposed) vertex
      A vertex not yet matched. Augmenting always starts and ends at a free vertex (one on each side).
      Alternating path
      A path whose edges alternate between not in the matching and in the matching.
      Augmenting path
      An alternating path that starts and ends at free vertices. It has one more non-matching edge than matching edges, so flipping it (swap in/out of the matching) increases the matching size by exactly one.
      Berge's theorem
      A matching is maximum if and only if it admits no augmenting path. This is why "augment until none remains" is correct, and why greedy (which never augments) can stop short.
      Greedy matching
      GreedyMaximumCardinalityMatching (unsorted): walk the vertices; whenever both endpoints of an edge are still free, add it. Fast O(E), always maximal, but only a ½-approximation.
      Augmenting-path method
      From each free left vertex, search (DFS) for a single augmenting path and flip it. Repeat until none remain. This is the classic Hungarian / Kuhn bipartite matching, running in O(V·E). The conceptual core of Hopcroft-Karp.
      Hopcroft-Karp
      HopcroftKarpMaximumCardinalityBipartiteMatching: a BFS finds the length of the shortest augmenting path and layers the graph; a DFS then extracts all vertex-disjoint shortest augmenting paths and flips them at once. Only O(√V) such phases are needed, giving O(E·√V). It also begins from a greedy warm-start matching.
      Why batching helps
      Augmenting one path at a time can take up to V phases. Hopcroft-Karp proves that after each batch the shortest augmenting path gets strictly longer, capping the number of phases at √V — far fewer searches on large graphs.
      What you're watching — three matching algorithms

      Pick any two algorithms with the A and B selectors

      Three maximum-cardinality bipartite matching strategies, two of them JGraphT classes from org.jgrapht.alg.matching:

      • Greedy — GreedyMaximumCardinalityMatching. Adds any conflict-free edge it meets. Stops at a maximal matching that may be far from maximum (see the greedy-trap example: it finds 2 where the maximum is 4).
      • Augmenting paths — the classic Kuhn / Hungarian method: find one augmenting path at a time and flip it. Always reaches the maximum; the building block Hopcroft-Karp optimises.
      • Hopcroft-Karp — HopcroftKarpMaximumCardinalityBipartiteMatching. Warm-starts greedily, then in each phase finds the shortest augmenting path length by BFS and flips all disjoint shortest paths together. Reaches the same maximum in far fewer phases.

      The punchline: greedy's matching size can be strictly smaller than the other two, which always agree on the true maximum. Compare the phase and augmenting-path counts to see why Hopcroft-Karp scales better. The two panes run independently; total progress is the slower pane's step count.
