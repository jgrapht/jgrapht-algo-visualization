      Glossary — spanning trees, union-find, cut property, cycle property
      Spanning tree
      A subset of edges that connects every vertex with no cycles. A connected graph on n vertices has spanning trees with exactly n − 1 edges. The minimum spanning tree (MST) minimises total edge weight.
      Spanning forest (MSF)
      If the graph is disconnected, no single tree can span it. All three algorithms instead return one minimum tree per connected component — a minimum spanning forest with n − (number of components) edges.
      Union-find (disjoint-set)
      Tracks which vertices are already in the same partial tree. find(v) returns v's component representative; union(a,b) merges two components. Kruskal and Borůvka use it; node tints on the canvas are union-find sets. JGraphT uses union by rank with path compression.
      Cut property
      For any partition of the vertices into two sides, the cheapest edge crossing the cut is in some MST. This is why a "cheapest safe edge" can always be added — it is the engine behind Prim and Borůvka.
      Cycle property
      The most expensive edge on any cycle is in no MST. Kruskal's cycle check is this property in action: an edge whose endpoints are already connected would be the last (hence heaviest-so-far) edge of a cycle, so it is rejected.
      Kruskal's algorithm
      Sort all edges by weight ascending; scan them cheapest-first. Add an edge unless its endpoints are already in the same component (that would make a cycle). O(E log E), dominated by the sort.
      Prim's algorithm
      Grow a single tree from a start vertex. Keep a priority queue of every vertex keyed by the cheapest known edge connecting it to the tree; repeatedly extract the minimum, splice in that edge, and relax its neighbours. JGraphT uses a Fibonacci heap: O(E + V log V).
      Borůvka's algorithm
      In each round, every component simultaneously selects its own cheapest outgoing edge; all selected edges are added at once and components merge. The number of components at least halves per round, so O((E+V) log V) in O(log V) rounds.
      Relaxation (Prim)
      When a vertex is added to the tree, each neighbour q outside the tree is checked: if the new connecting edge is cheaper than q's current best, update q's key (a decrease-key). The "relaxations" counter tallies these improvements.
      Tie-breaking
      When several edges share a weight, the MST is no longer unique — but its total weight is. The three algorithms may pick different equal-weight edges (watch the ties example). Borůvka breaks ties by a fixed edge order to stay deterministic and avoid cycles within a round.
      What you're watching — three MST algorithms

      Pick any two algorithms with the A and B selectors

      Three minimum-spanning-tree algorithms from JGraphT's org.jgrapht.alg.spanning package are wired up, each animated step-by-step on the same weighted undirected graph:

      • Kruskal — KruskalMinimumSpanningTree. Sorts the whole edge set by weight, then walks it cheapest-first, using union-find to skip any edge that would close a cycle. You see edges light up in global weight order, scattered all over the graph.
      • Prim — PrimMinimumSpanningTree. Grows a single connected tree out from a start vertex. The indigo frontier edges are the current cheapest way to attach each outside vertex; the orange vertex is the one just extracted from the heap.
      • Borůvka — BoruvkaMinimumSpanningTree. Works in rounds: every component picks its own cheapest outgoing edge (all shown in cyan at once), then they are committed together and components merge. Watch the component count fall each round.

      The punchline: on a graph with distinct weights, all three end on the identical edge set and identical total weight — but the order in which edges are added (the numbered result list below) is completely different. On the ties example they may even pick different edges, yet the totals still match. On the disconnected example all three degrade gracefully to a minimum spanning forest.
